# Backtesting Plan

Backtesting is intentionally not implemented in Phase 1–2.

## What Phase 1–2 does

Phase 1–2 creates the contracts and checks that later backtesting will rely on:
- actuals contracts
- run artifacts
- versioning fields
- fail-closed checks
- deterministic fixtures

## Future backtesting requirements

- rolling-window evaluation
- leakage prevention
- blocked game accounting
- baseline comparisons
- versioned run artifacts
- explicit error metrics
